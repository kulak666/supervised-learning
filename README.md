# Обучение с учителем: качество модели

## Описание проекта
Интернет-магазин «В один клик» продаёт разные товары: для детей, для дома, мелкую бытовую технику, косметику и даже продукты. Отчёт за прошлый период показал, что активность покупателей начала снижаться. Привлекать новых клиентов уже не так эффективно: о магазине и так знает большая часть целевой аудитории.

Нужно разработать решение, которое позволит персонализировать предложения постоянным клиентам, чтобы увеличить их покупательскую активность.

## Данные
Данные предоставлены в табличной форме и состоят из нескольких файлов.

### `market_file.csv`
Содержит данные о поведении покупателя на сайте, коммуникациях с покупателем и продуктовом поведении.

**Поля:**
- `id` — номер покупателя в корпоративной базе данных  
- `Покупательская активность` — целевой признак: «снизилась» или «прежний уровень»  
- `Тип сервиса` — уровень сервиса, например «премиум» и «стандарт»  
- `Разрешить сообщать` — можно ли присылать покупателю дополнительные предложения  
- `Маркет_актив_6_мес` — среднемесячное значение маркетинговых коммуникаций за последние 6 месяцев  
- `Маркет_актив_тек_мес` — количество маркетинговых коммуникаций в текущем месяце  
- `Длительность` — сколько дней прошло с момента регистрации на сайте  
- `Акционные_покупки` — среднемесячная доля покупок по акции за последние 6 месяцев  
- `Популярная_категория` — самая популярная категория товаров за последние 6 месяцев  
- `Средний_просмотр_категорий_за_визит` — сколько категорий в среднем просмотрено за визит (последний месяц)  
- `Неоплаченные_продукты_штук_квартал` — число неоплаченных товаров в корзине за последние 3 месяца  
- `Ошибка_сервиса` — число сбоев, затронувших покупателя во время посещения сайта  
- `Страниц_за_визит` — среднее число страниц за визит (последние 3 месяца)

### `market_money.csv`
Данные о выручке, которую получает магазин с покупателя (сколько покупатель потратил за период взаимодействия с сайтом).

**Поля:**
- `id` — номер покупателя  
- `Период` — период (например, `текущий_месяц`, `предыдущий_месяц`)  
- `Выручка` — сумма выручки за период  

### `market_time.csv`
Данные о времени (в минутах), которое покупатель провёл на сайте в течение периода.

**Поля:**
- `id` — номер покупателя  
- `Период` — период  
- `минут` — время на сайте (мин)  

### `money.csv`
Данные о среднемесячной прибыли покупателя за последние 3 месяца.

**Поля:**
- `id` — номер покупателя  
- `Прибыль` — прибыль магазина от покупателя  

## Цель проекта
Разделить задачу на два этапа:

1. Разработать модель, которая предскажет вероятность снижения покупательской активности.
2. Выделить сегмент покупателей, проанализировать его и предложить, как увеличить покупательскую активность, используя:
   - результаты моделирования,
   - данные о прибыли покупателей,
   - исходные данные (при необходимости).

## Идеи сегментации
Возможные сегменты:
- Клиенты с максимальной долей покупок по акции и высокой вероятностью снижения активности  
- Клиенты, покупающие только технику (товары с длинным жизненным циклом)  
- Клиенты, покупающие товары из категории «Товары для себя» (новая категория на основе текущих) или «Товары для детей»  
- Клиенты с высокой вероятностью снижения активности и наиболее высокой прибыльностью
